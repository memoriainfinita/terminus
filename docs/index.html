Instalación y Configuración

Método 1: CDN (Recomendado)

<!-- CSS + JS desde CDN -->
<link rel="stylesheet" href="https://cdn.jsdelivr.net/gh/memoriainfinita/terminus@latest/docs/dist/terminal.min.css">
<script src="https://cdn.jsdelivr.net/gh/memoriainfinita/terminus@latest/docs/dist/terminal.min.js" defer></script>

<!-- Terminal básico -->
<div class="gnu-terminal"
     data-theme="dark"
     data-prompt="gnu$"
     data-commands='{"help":"Lista comandos","about":"Proyecto GNU"}'></div>

Ventajas: Auto-actualización, CDN rápido, cero configuración

Método 2: NPM

npm install @memoriainfinita/terminus
# o
yarn add @memoriainfinita/terminus

Ventajas: Control de versiones, bundling optimizado

<script src="./js/terminal.min.js" defer></script>

Ventajas: Sin dependencias externas, máximo control

Parámetro: data-theme

Valor Descripción Colores Uso Recomendado
"dark" (default) Tema oscuro profesional Gris/Verde General, producción
"matrix" Verde clásico estilo Matrix Negro/Verde Demos, cyberpunk
"ocean" Azul profundo elegante Azul/Cyan Corporativo, tech
"amber" Dorado retro terminal Negro/Amber Vintage, sistemas legacy

Parámetros de Configuración

Atributo Tipo Default Descripción Ejemplo
data-prompt String "user@host:~$" Personaliza el prompt del terminal "admin@server:~#"
data-welcome String null Mensaje inicial al cargar "Sistema iniciado"
data-commands JSON {} Comandos personalizados {"help":"Lista comandos"}
data-typing-speed Number 50 Velocidad escritura (ms) 30 (más rápido)
data-auto-focus Boolean true Auto-focus al cargar "false"

Comandos JSON: Estructura Avanzada

{
  "help": "Lista todos los comandos disponibles",
  "about": "Información sobre este terminal",
  "clear": "Limpia la pantalla",
  "status": {
    "description": "Estado del sistema",
    "response": "Sistema operativo: Online\nCPU: 45%\nRAM: 2.1GB/8GB"
  },
  "contact": {
    "description": "Información de contacto",
    "response": "Email: admin@example.com\nWeb: https://example.com"
  }
}

Tip: Los comandos pueden ser strings simples o objetos con description + response para mayor control

Consideraciones de Seguridad

🛡️ Importante:

  • Sanitiza el contenido JSON antes de usar en producción
  • Evita incluir información sensible en data-commands
  • Valida comandos del lado servidor si conectas a APIs
  • Usa HTTPS siempre para evitar inyección de código

Preguntas FrecuentesFAQ

Respuestas a las consultas más comunes sobre TERMINUS

¿Qué tan fácil es implementar TERMINUS?

+

Solo 2 líneas de código: CSS y JS. Auto-inicialización automática, sin configuración adicional. Literalmente copias dos archivos y funciona inmediatamente.

¿Puedo personalizar temas y colores?

+

4 temas predefinidos (matrix, ocean, amber, dark) + temas personalizados via CSS variables. Control total sobre colores, fuentes y animaciones.

¿Cómo agregar comandos personalizados?

+

Atributo data-commands con JSON o configurador interactivo incluido. Soporte para comandos simples, objetos complejos y respuestas dinámicas.

¿Es compatible con móviles?

+

Completamente responsive. Funciona en desktop, tablet y móvil con teclado virtual optimizado y gestos táctiles incluidos.

¿Necesita dependencias externas?

+

Cero dependencias. Vanilla HTML/CSS/JS puros. Total: 5.1 KB minificado. No requiere jQuery, React ni ninguna librería externa.

¿Cómo distribuir en producción?

+

CDN jsDelivr, GitHub Pages, descarga directa o npm. Archivos minificados incluidos. Compatible con bundlers como Webpack y Vite.

¿Está probado y es confiable?

+

Completamente verificado y estable. Todas las funcionalidades probadas en múltiples navegadores y dispositivos. Usado en producción.

¿Compatible con React, Vue, Angular?

+

Sí, vanilla JavaScript se integra con cualquier framework. Solo asegurar DOM cargado. Incluye ejemplos para todos los frameworks populares.

¿Cuál es la licencia?

+

GNU GPL v3 - libre para uso personal y comercial. Open source en GitHub. Código fuente completo disponible para auditoría y contribuciones.
